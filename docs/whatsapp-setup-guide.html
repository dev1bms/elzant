🧩 التجهيز

1

تجهيز حساب Meta وتطبيق واتساب

  1. ادخل إلى developers.facebook.com وسجّل الدخول بحساب فيسبوك.
  2. أنشئ Business Portfolio (حساب أعمال) إن لم يكن لديك، من business.facebook.com.
  3. أنشئ تطبيقاً جديداً: Create App ← اختر نوع Business.
  4. داخل التطبيق: Add Product ← أضِف WhatsApp.
  5. سيُنشئ لك واتساب رقم اختبار (Test number) وWhatsApp Business Account تلقائياً — كافٍ للتجربة قبل ربط رقمك الحقيقي.
نصيحة: اضبط اسم العرض للنشاط (Display name) باسم مناسب مثل «زفاف محمود ورينان» أو اسم العائلة — هذا ما يراه المدعو كمُرسِل.
2

اجمع القيم الستّ المطلوبة

كل هذه القيم تُدخَل لاحقاً في لوحة الأدمن (لا حاجة لأي ملف .env):

القيمةمن أين تجدها في Meta
Phone Number IDWhatsApp ← API Setup: تحت الرقم المُختار.
WhatsApp Business Account IDنفس صفحة API Setup (WABA ID).
Access Token (التوكن)توكن مؤقت في API Setup للتجربة؛ للإنتاج أنشئ System User بتوكن دائم من إعدادات الأعمال.
App SecretApp ← Settings → Basic → App Secret (اضغط Show).
Verify Tokenسلسلة عشوائية من عندك (اختَرها الآن) — ستُدخِلها في مكانين متطابقين: الأدمن وMeta.
Graph API Versionاختياري — الافتراضي v21.0.
التوكن الدائم: التوكن المؤقت ينتهي خلال ساعات. للإنتاج استخدم توكن System User دائم الصلاحية حتى لا يتوقّف الإرسال فجأة.
3

أنشئ قالب الدعوة (Utility)

الرسائل التي يبدؤها العمل تتطلّب قالباً معتمداً من Meta. من WhatsApp Manager → Message Templates → Create Template:

  • الفئة (Category): اختر Utility — أرخص بكثير وغالباً مجاني داخل نافذة 24 ساعة (تجنّب Marketing).
  • اللغة: العربية ar.
  • Body: الصق أحد النماذج الجاهزة أدناه؛ المتغيّر {{1}} = اسم المدعو.
  • زر (Button) نوع URL ديناميكي: الأساس https://elzant.com/i/ والمتغيّر {{1}} = توكن الدعوة. الرابط النهائي يصبح https://elzant.com/i/<token>/.
  • احفظ واسم القالب (مثل invite_formal_ar) وانتظر الاعتماد (قد يستغرق دقائق إلى أيام).
جاهزة لك: النماذج مزروعة مسبقاً في الأدمن ضمن «قوالب واتساب» — انسخ نصّها من قسم النماذج الجاهزة والصقه في Meta كما هو.

⚙️ الإعداد داخل الموقع

4

أدخِل القيم في لوحة الأدمن

افتح /admin/ ← «إعدادات واتساب» واملأ:

  • Cloud API: Phone Number ID · WhatsApp Business Account ID · إصدار Graph API.
  • القالب الافتراضي: اسم القالب المعتمد · لغته (ar) · رمز الدولة الافتراضي (مصر=20).
  • الأسرار (مقنّعة، superuser فقط): Access Token · App Secret · Verify Token.
الحقول المقنّعة: تظهر فارغة لأسباب أمنية. أدخِل القيمة مرة واحدة واحفظ. عند أي تعديل لاحق، اترك حقل السرّ فارغاً للإبقاء على قيمته — لن يُمحى.
أمان: بما أن الأسرار صارت داخل قاعدة البيانات، احمِ ملف db.sqlite3 ونسخه الاحتياطية (صلاحيات ملف صارمة، ونسخ مشفّرة/بعيدة). شاشة الإعدادات محصورة بمستخدم superuser.
5

اضبط الـ Webhook في Meta

الـ Webhook هو ما يعيد إليك حالات التسليم والقراءة. من App ← WhatsApp → Configuration → Webhook:

Callback URLhttps://elzant.com/webhooks/whatsapp/
Verify Tokenنفس القيمة التي أدخلتها في «إعدادات واتساب» بالأدمن (متطابقة تماماً).
Subscribeفعّل الاشتراك في حقل messages (يغطّي sent / delivered / read / failed).
شرط: يجب أن يكون elzant.com عبر HTTPS (نفق Cloudflare) وضمن CSRF_TRUSTED_ORIGINS. الـ Webhook يتحقّق من توقيع HMAC-SHA256، ويرفض أي طلب غير موقّع بـ 403.

✅ الاختبار والتفعيل

6

أرسل رسالة اختبار

  1. في «إعدادات واتساب» أضِف «رقم اختبار الإرسال» (رقمك أنت) واحفظ.
  2. اضغط زر «إرسال رسالة اختبار».
  3. وأنت في الوضع الآمن (الإرسال الحيّ متوقّف): يُحاكي فقط دون إرسال فعلي أو تكلفة — للتأكّد من التدفّق.
ملاحظة: قبل التوثيق الكامل تسمح Meta بالإرسال إلى أرقام اختبار مُضافة فقط — أضِف رقمك في API Setup ← Recipients.
7

فعّل الإرسال الحيّ

  1. تأكّد أن القالب معتمد في Meta، وأن كل القيم مُدخلة.
  2. في «إعدادات واتساب» فعّل «تفعيل الإرسال الحيّ» واحفظ.
  3. أعِد زر «إرسال رسالة اختبار» — هذه المرة يصلك فعلياً على واتساب.
  4. افتح رابط الدعوة من الرسالة، وتأكّد من انتقال الحالة في اللوحة إلى «سُلّمت» ← «قُرئت» ← «فُتح الرابط».
أُرسلت← سُلّمت← قُرئت← فُتح الرابط← كتب تهنئة

👥 الاستخدام

8

أنشئ حسابات المُرسِلين (العائلة)

  1. في /admin/ ← Users: أنشئ مستخدماً لكل فرد من العائلة (اسم مستخدم + كلمة مرور). فعّل Staff status.
  2. في «المُرسِلون (العائلة)» (InviterProfile): اربط المستخدم، واكتب الاسم الظاهر واختر الجهة (عريس/عروس).
  3. فعّل «يرى كل المدعوين» لمن يحتاج عرض «الكل + من دعا كلاً» (غالباً منظّم واحد لكل عائلة).
الوصول: من لا يملك InviterProfile (ولا صلاحية superuser) لا يستطيع دخول اللوحة /panel/.
9

الإرسال من اللوحة العائلية

  1. يفتح الفرد /panel/ ويسجّل الدخول باسمه وكلمة مروره.
  2. «دعوة جديدة»: يُدخل الاسم + رقم الهاتف، ثم إرسال — تُرسَل الدعوة آلياً برابطها الخاص.
  3. إن كان الرقم مدعوّاً مسبقاً: تظهر رسالة «مدعو بواسطة فلان»، ولا يُرسَل تكرار إلا بعد تأكيد.
  4. يرى كلٌّ دعواته وحالاتها؛ والمصرّح له يبدّل إلى «كل المدعوين» لرؤية من دعا كلاً.

⚠️ تحذيرات مهمّة

!

قبل الإرسال الواسع

  • التكلفة: الفوترة لكل رسالة (منذ 1 يوليو 2025) ويحدّدها رمز دولة المُستقبِل. قوالب Utility أرخص وغالباً مجانية داخل نافذة 24 ساعة.
  • الموافقة (Opt-in): سياسة واتساب تتطلّب أن يكون المستقبِل موافقاً على التلقّي. أرسل لمدعوّين معروفين للعائلة فقط.
  • جودة الرقم (Quality Rating): ابدأ تدريجياً وراقب التقييم في WhatsApp Manager — البلاغات المتكرّرة قد تُقيّد الرقم أو تحظره.
  • الرقم المُرسِل رقم أعمال لا رقم شخصي — لذلك اضبط اسم العرض بوضوح ليعرفه المدعو.
